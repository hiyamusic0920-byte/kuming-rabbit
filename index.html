苦命兔 QA 日誌｜能力地圖

這不是單純的漫畫專案，而是一份 AI Native QA Portfolio。
透過 15 話漫畫，記錄我如何把 AI 從「幫忙產報告的工具」，逐步治理成能參與 QA 交付、知識累積、多 Agent 協作與流程評估的工作系統。

  1. 第01話

    讓 AI 幫我做測試報告

    展示能力：QA 報告自動化

    對應產出 / 方法：pytest、Playwright、PDF Report、QA/UAT Report
  2. 第02話

    報告有了，但看完更想哭

    展示能力：Ticket Analysis / Context Engineering

    對應產出 / 方法：Redmine / Jira / Sub-task 資訊整合、範圍差異、邏輯缺口
  3. 第03話

    拯救苦命兔大作戰

    展示能力：長期知識累積

    對應產出 / 方法：pages/_registry、selector 知識、互動陷阱、回饋機制
  4. 第04話

    跨 Agent 合作挑戰

    展示能力：Multi-Agent Handoff

    對應產出 / 方法：PROJECT_RULES、handoff、Claude / Codex 共用規則
  5. 第05話

    200 個 E2E

    展示能力：Test Strategy / Prompt Chaining

    對應產出 / 方法：test-plan → TC → script generator，決定什麼值得 E2E
  6. 第06話

    苦命兔超努力，無效的那種

    展示能力：Flaky Test Governance

    對應產出 / 方法：retry 規則、flaky 標記、失敗分析、停止條件
  7. 第07話

    順手把整個房間打掃了

    展示能力：Permission Boundary / Safe Automation

    對應產出 / 方法：禁止行為、檔案操作邊界、先詢問再改
  8. 第08話

    被罵過一次後

    展示能力：Feedback Loop / Agent Behavior Tuning

    對應產出 / 方法：回饋機制、避免過度懲罰導致 Agent 不敢決策
  9. 第09話

    重新檢視工作流程

    展示能力：Tacit Knowledge Extraction

    對應產出 / 方法：把老鳥經驗轉成文件、規則、流程與 QA heuristics
  10. 第10話

    失之毫里，差之千里

    展示能力：Requirements Audit

    對應產出 / 方法：歧義偵測、多重解讀、需求確認前置
  11. 第11話

    辯論大賽

    展示能力：Cross-Agent Consensus

    對應產出 / 方法：Claude / Codex Review、文件分類決策框架
  12. 第12話

    秘密小本本

    展示能力：Single Source of Truth

    對應產出 / 方法：共同規則、Agent Governance、文件標準化
  13. 第13話

    那隻兔子就像渣男

    展示能力：Decision Checkpoint Design

    對應產出 / 方法：AI Understanding Mirror、Audit Gate、規則觸發機制
  14. 第14話

    偷偷改家規

    展示能力：Traceability / Change Management

    對應產出 / 方法：Change Log、Work Log、Commit History、Process Audit
  15. 第15話

    吃你的狗食

    展示能力：Evaluation / Dogfooding

    對應產出 / 方法：Retro、Eval Framework、重跑失敗票驗證改善效果
